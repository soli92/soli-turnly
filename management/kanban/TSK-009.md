# TSK-009 — Audit log persistente + job asincroni

## Obiettivo
Implementare la scrittura sistematica dell'audit log per ogni azione critica, e i
background job (Inngest) per: generazione turni da ricorrenza, notifiche email,
e pulizia sessioni scadute.

### Helper (lib/audit.ts)
```typescript
import { db } from '@/lib/db';
import { auditLogs } from '@/lib/db/schema';

export interface AuditEntry {
  actorId: string;
  action: AuditAction;
  entityType: 'shift' | 'request' | 'user' | 'absence' | 'coverage';
  entityId: string;
  before?: unknown;
  after?: unknown;
  ip?: string;
  userAgent?: string;
}

export type AuditAction =
  | 'shift.create' | 'shift.update' | 'shift.delete'
  | 'request.create' | 'request.approve' | 'request.reject' | 'request.cancel'
  | 'user.create' | 'user.update' | 'user.password_change'
  | 'absence.create' | 'absence.delete'
  | 'swap.propose' | 'swap.accept' | 'swap.admin';

export async function insertAuditLog(entry: AuditEntry) {
  await db.insert(auditLogs).values({
    ...entry,
    before: entry.before ? JSON.stringify(entry.before) : null,
    after: entry.after ? JSON.stringify(entry.after) : null,
    createdAt: new Date(),
  });
}
```

### Azioni che DEVONO loggare (T-SEC-05)
- Ogni modifica a shift, request, user, absence
- Ogni approvazione/rifiuto richiesta
- Ogni login/logout (Auth.js events)
- Ogni cambio password

### Job: generateRecurringShifts (lib/jobs/generateRecurringShifts.ts)
```typescript
export const generateRecurringShifts = inngest.createFunction(
  { id: 'generate-recurring-shifts' },
  { event: 'shift.recurrence.trigger' },
  async ({ event, step }) => {
    const { recurrenceId, periodStart, periodEnd } = event.data;

    const recurrence = await step.run('load-recurrence', () =>
      db.query.recurrences.findFirst({ where: eq(recurrences.id, recurrenceId) })
    );

    const dates = await step.run('expand-dates', () =>
      expandRecurrence(recurrence!, periodStart, periodEnd)
    );

    // RB-11: skip assenze e festivi
    const validDates = await step.run('filter-conflicts', () =>
      filterAbsencesAndHolidays(dates, recurrence!.userId)
    );

    await step.run('create-shifts', () =>
      db.insert(shifts).values(
        validDates.map(d => buildShiftFromRecurrence(recurrence!, d))
      )
    );
  }
);
```

### Job: sendNotificationEmail
- Trigger: evento `notification.email.send` con { to, subject, template, data }
- Template Handlebars in `lib/emails/` (shift-assigned, request-approved, etc.)
- Provider: Resend (o Nodemailer in dev)

### Job: cleanExpiredSessions
- Cron: ogni notte alle 02:00
- Rimuove sessioni Auth.js scadute dalla tabella `sessions` (se strategy: database)

## Criteri di accettazione
