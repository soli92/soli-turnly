# TSK-028 — Centro notifiche

## Obiettivo

Implementare la pagina dedicata "Centro notifiche" (RF-N, screen 21 inventario) accessibile
a tutti i ruoli (sia admin sia dipendente). Potenziare `NotificationBell` con link "Vedi
tutte" che naviga al centro notifiche. La campanella è già scaffoldata a
`components/notifications/NotificationBell.tsx`.

Le API sono già presenti:
- `GET /api/notifications` — lista con paginazione (TSK-004/TSK-008)
- `PATCH /api/notifications/[id]/read` — mark as read (TSK-008)

## Componenti / Endpoint da creare

### Pagine (fe)

```
app/(employee)/notifications/
  page.tsx                           ← RSC: verifica sessione (tutti i ruoli); metadata
  _components/
    NotificationCenterClient.tsx     ← Client: TanStack Query lista completa, paginazione
    NotificationItem.tsx             ← singola notifica: titolo, body, timestamp, badge "Non letta", link entità
    NotificationEmptyState.tsx       ← empty state: "Nessuna notifica" con icona
    MarkAllReadButton.tsx            ← pulsante "Segna tutte come lette" (PATCH batch)
```

Nota: la pagina è sotto `(employee)/` ma sia admin che dipendente vi accedono — il layout
`(employee)/layout.tsx` permette entrambi i ruoli (controlla solo autenticazione, non il
ruolo). Per l'admin aggiungere link nel nav admin in `(admin)/layout.tsx`.

### Endpoint aggiuntivo (be)

```
PATCH /api/notifications/read-all
```

- Segna tutte le notifiche `WHERE userId = session.user.id AND readAt IS NULL` come lette
- Risponde `{ updated: N }`

### NotificationBell — potenziamenti

```
components/notifications/NotificationBell.tsx  ← aggiungere:
  - link "Vedi tutte →" in footer Popover → naviga a /notifications
  - pulsante "Segna tutte come lette" nel Popover header
```

### NotificationItem — link entità

Ogni notifica ha `relatedEntityType` + `relatedEntityId`; mapping in link:

```typescript
// entity routing
const entityLink = {
  request:  `/requests/${n.relatedEntityId}`,
  shift:    `/calendar`,
  absence:  `/requests`,
};
```

## Acceptance Criteria

- [ ] RF-N CA1: dopo che un dipendente invia una richiesta, l'admin vede notifica nel centro notifiche senza refresh manuale (SSE invalida la query)
- [ ] RF-N CA2: dopo approvazione/rifiuto, il dipendente richiedente vede notifica con esito
- [ ] RF-N CA3: un dipendente non vede notifiche di altri utenti — `GET /api/notifications` filtra sempre per `userId = session.user.id`
- [ ] Paginazione: `limit=20` di default; "Carica altre" aggiunge la pagina successiva (infinite scroll o pulsante)
- [ ] `MarkAllReadButton`: click → `PATCH /api/notifications/read-all` → badge non lette torna a 0 sia in pagina sia in NotificationBell
- [ ] NotificationItem non letta: sfondo `indigo-50`; click → segna come letta + naviga all'entità correlata (se presente)
- [ ] Link "Vedi tutte →" nel Popover Bell naviga correttamente a `/notifications`
- [ ] Empty state: quando `data=[]` mostra `NotificationEmptyState` (nessun errore visivo)
- [ ] WCAG 2.2 AA: lista con `role="list"`, ogni item con `role="listitem"`, timestamp leggibile con `<time datetime="ISO">`
- [ ] Layout mobile 375px: item leggibili, nessun testo troncato in modo illeggibile
