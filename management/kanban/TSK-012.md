# TSK-012 — Gate qualità codice

## Obiettivo
Configurare e far passare il gate di qualità finale: TypeScript strict, ESLint,
Prettier, test coverage ≥ 80% su `lib/rules/`, e review CQRL v2.12 via `/review`.

## TypeScript strict

```json
// tsconfig.json — opzioni strict da verificare
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "noImplicitOverride": true
  }
}
```
- `tsc --noEmit` deve passare con 0 errori
- Nessun `@ts-ignore` o `any` non giustificato nel codice produttivo

## ESLint config

```js
// eslint.config.mjs
import next from '@next/eslint-plugin-next';
import typescript from '@typescript-eslint/eslint-plugin';

export default [
  ...next.configs.recommended,
  {
    plugins: { '@typescript-eslint': typescript },
    rules: {
      '@typescript-eslint/no-explicit-any': 'error',
      '@typescript-eslint/no-floating-promises': 'error',
      '@typescript-eslint/no-misused-promises': 'error',
      'no-console': ['warn', { allow: ['error', 'warn'] }],
    },
  },
];
```

```typescript
        lines: 80,

```

## CI pipeline completa

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm' }
      - run: npm ci

      - name: TypeScript
        run: npx tsc --noEmit

      - name: Lint
        run: npx eslint .

      - name: Format check
        run: npx prettier --check .

      - name: Unit tests + coverage
        run: npx vitest run --coverage

      - name: E2E
        run: npx playwright test
        env: { CI: true }

      - name: A11y
        run: npx playwright test tests/a11y/
```
