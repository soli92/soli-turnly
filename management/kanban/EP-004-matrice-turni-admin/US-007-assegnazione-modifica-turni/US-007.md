# US-007 — Assegnazione, modifica e rimozione turni con validazione in tempo reale

## Descrizione
Come admin, voglio assegnare, modificare e rimuovere turni sulle celle della matrice con
feedback immediato sulle violazioni delle regole di business, affinché la pianificazione
sia sempre valida e io possa correggere i problemi prima del salvataggio.

## Business Rules
- RB-01: sovrapposizione turni per lo stesso dipendente → BLOCCANTE (impedisce salvataggio).
- RB-02: riposo < minimo (default 11h) tra turni adiacenti → AVVISO (richiede conferma esplicita).
- RB-04: più di maxGiorniConsecutivi (default 6) giorni lavorativi → AVVISO.
- RB-05: ore settimanali oltre oreSettSoft (48h) → AVVISO; oltre oreSettHard (60h) → BLOCCANTE.
- RB-08: turno durante un'assenza attiva → BLOCCANTE (impedisce salvataggio).
- RB-09: modifica/creazione nel passato → AVVISO (configurabile a BLOCCANTE).
- RB-15: turno in finestra di indisponibilità dichiarata → AVVISO (configurabile a BLOCCANTE).
- La validazione mostrata in UI e quella applicata al salvataggio lato server coincidono (RF-D CA5, T-INT-01).

## UI Reference
Schermata #5 Editor turno (modale su cella): selettore tipologia, orari custom, note, pannello avvisi/violazioni.

## Acceptance Criteria
- [ ] Assegnare un secondo turno sovrapposto allo stesso dipendente nello stesso giorno è impedito (blocco UI + 4xx API, T-DOM-02, RF-D CA1).
- [ ] Assegnare un turno con riposo < 11h verso il turno adiacente mostra un avviso inline con il dettaglio e richiede conferma prima del salvataggio (T-DOM-03, RF-D CA2).
- [ ] Assegnare un turno a un dipendente in assenza in quella data è impedito (T-DOM-04, RF-D CA3).
- [ ] Assegnare un turno in una finestra di indisponibilità dichiarata mostra un avviso (RF-D CA4, T-REQ-05).
- [ ] La stessa assegnazione che la UI segnala come bloccante è rifiutata anche dall'API con lo stesso motivo (T-INT-01).
- [ ] Una cella con avviso è evidenziata in giallo; una cella con violazione bloccante è evidenziata in rosso.
- [ ] L'admin può rimuovere un turno dalla cella; la rimozione rispetta le stesse regole di validazione.
- [ ] L'admin può inserire orari custom al posto di una tipologia preconfigurata.
