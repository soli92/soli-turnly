# TSK-007 — Form frontend con validazione condivisa

## Obiettivo
Implementare tutti i form dell'area admin e dipendente con React Hook Form + Zod,
usando gli stessi schema del BE per garantire parità UI↔API (T-INT-01).

## Form da implementare

### Area Admin
- `ShiftEditorForm` — crea/modifica turno (usato da TSK-005 modal)
- `AbsenceForm` — registra assenza (tipo + date range)
- `RecurrenceWizard` — 3 step (tipo → sequenza → anteprima)
- `SwapForm` — selezione turno A + turno B + anteprima impatto
- `CoverageForm` — fabbisogno per fascia/qualifica
- `UserForm` — crea/modifica dipendente (tutti i campi, admin only)
- `ApprovalPanel` — approva/rifiuta richiesta con anteprima impatto

### Area Dipendente
- `RequestForm` — step 1 selezione tipo → step 2 form dinamico
  - Assenza: tipo radio + date range
  - Scambio: selezione turno collega
  - Nuovo turno: data + tipologia
  - Modifica turno: turnoId + modifica proposta
- `ProfileForm` — modifica solo campi consentiti (T-SEC-04)
- `AvailabilityForm` — finestre ricorrenti o date specifiche

## Schema Zod condiviso (lib/zod/)

```typescript
// lib/zod/shiftSchema.ts — USATO SIA DA FE CHE DA BE (T-INT-01)
export const shiftCreateSchema = z.object({
  userId: z.string().uuid(),
  shiftTypeId: z.string().uuid().optional(),
  date: z.string().date(),
  startDt: z.string().datetime({ offset: true }),
  endDt: z.string().datetime({ offset: true }),
  notes: z.string().max(500).optional(),
}).refine(
  (data) => new Date(data.endDt) > new Date(data.startDt),
  { message: 'end_time deve essere dopo start_time', path: ['endDt'] }
);

// lib/zod/userPatchSchema.ts — SOLO campi consentiti al dipendente (RB-13)
export const userSelfPatchSchema = z.object({
  phone: z.string().max(20).optional(),
  contactEmail: z.string().email().optional(),
  contactPreferences: z.string().optional(),
  // qualificationId, contractHours, etc. → ASSENTI = non modificabili
});
```

## Integrazione TanStack Query

```typescript
const createShiftMutation = useMutation({
  mutationFn: (data: ShiftCreate) =>
    fetch('/api/shifts', { method: 'POST', body: JSON.stringify(data) }).then(r => r.json()),
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['shifts', week] });
    toast.success('Turno salvato');
  },
  onError: (err: RuleViolationError) => {
    // Mapping errori BE → field errors React Hook Form
    err.issues?.forEach(issue => form.setError(issue.path[0], { message: issue.message }));
  },
});
```

## Criteri di accettazione
- [ ] Zod schema FE e schema BE sono identici (stesso file importato da entrambi) — T-INT-01
- [ ] Errori di validazione FE: messaggi in italiano, posizionati sul campo corretto
- [ ] Errori API (400): mappati automaticamente ai field errors del form
- [ ] ProfileForm: campi contrattuali non renderizzati / in sola lettura — T-SEC-04
- [ ] RequestForm step 2 "Scambio": mostra solo turni di colleghi della stessa fascia
- [ ] ApprovalPanel: mostra pannello impatto (violazioni + warnings) prima di approvare
