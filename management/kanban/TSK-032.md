# TSK-032 — Setup Neon DB produzione

## Obiettivo

Creare il database PostgreSQL managed su Neon per l'ambiente di produzione,
eseguire le migration e seminare l'utente admin iniziale.

## Step

### 1. Creare il progetto su Neon

- Accedere a [neon.tech](https://neon.tech) → New project
- Nome progetto: `soli-turnly`
- Regione: `eu-west-1` (Francoforte) o la più vicina
- Recuperare due connection string dalla dashboard:
  - **Pooled** (per l'app): `DATABASE_URL`
  - **Direct** (per le migration): `DATABASE_URL_UNPOOLED`

### 2. Eseguire le migration su prod

```bash
cd code/app
DATABASE_URL="<DIRECT_URL>" npx drizzle-kit migrate
```

Verificare che tutte le migration di `db/migrations/` vengano applicate senza errori.

### 3. Seed utente admin iniziale

```bash
cd code/app
DATABASE_URL="<DIRECT_URL>" npx tsx db/seed.ts
```

Annotare le credenziali dell'admin generato dal seed (email + password temporanea).

### 4. Verificare connessione

```bash
# Quick check: contare le tabelle create
DATABASE_URL="<DIRECT_URL>" npx tsx -e "
import { db } from './db/index';
import { users } from './db/schema';
const r = await db.select().from(users);
console.log('Users:', r.length);
process.exit(0);
"
```

## Acceptance Criteria

- [ ] Progetto Neon creato e raggiungibile
- [ ] Tutte le migration applicate (nessun errore `drizzle-kit migrate`)
- [ ] Almeno 1 utente admin presente nella tabella `users`
- [ ] `DATABASE_URL` (pooled) e `DATABASE_URL_UNPOOLED` (direct) disponibili per TSK-033
