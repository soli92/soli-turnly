# TSK-015 — Tipologie di turno page

## Obiettivo

Implementare la pagina di gestione delle tipologie di turno (RF-C): lista con
TanStack Table, form modal di creazione/modifica, disattivazione proteggendo
l'integrità referenziale.

## Componenti da creare

```
app/(admin)/shift-types/page.tsx            ← RSC: fetch lista tipologie
components/shift-types/
  ShiftTypeTable.tsx                         ← TanStack Table (nome, orari, durata, colore, stato)
  ShiftTypeModal.tsx                         ← shadcn Dialog + react-hook-form + Zod
  ShiftTypeColorPicker.tsx                   ← swatch di colori predefiniti
  ShiftTypeBadge.tsx                         ← pill colorato (usato anche in TSK-005)
```

### ShiftTypeModal — campi

| Campo | Tipo | Validazione |
|---|---|---|
| nome | text | obbligatorio, max 50 |
| startTime | time HH:mm | obbligatorio |
| endTime | time HH:mm | obbligatorio |
| attraversaMezzanotte | boolean | auto-derivato: `endTime < startTime` |
| durataMinuti | number readonly | calcolato live (RB-12: DST-safe con date-fns/tz) |
| pausaMinuti | number | default 0, ≥ 0 |
| colore | hex string | swatch picker |
| attiva | boolean | default true |

### Calcolo durata live (RF-C CA1, RB-12)

```typescript
// onChange startTime/endTime — usa date-fns/tz con Europe/Rome
const durataMin = calcShiftDuration(startTime, endTime); // da lib/rules/calculateShiftDuration
// Mostra warning se attraversaMezzanotte = true: "Turno notturno: fine il giorno successivo"
```

### Disattivazione vs eliminazione (RF-C CA2)

- Tipologia con turni associati: solo `active = false` (PATCH), non DELETE
- Tipologia senza turni: DELETE con conferma `AlertDialog`
- Badge "In uso" se esistono `shifts` che la referenziano
