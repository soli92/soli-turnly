# TSK-016 — Anagrafica dipendenti page

## Obiettivo

Implementare la pagina di gestione dipendenti (RF-B): lista filtrabile/ricercabile
con TanStack Table, form modal CRUD, disattivazione con conservazione storico turni.

## Componenti da creare

```
app/(admin)/staff/page.tsx                 ← RSC: fetch lista dipendenti
components/staff/
  StaffTable.tsx                            ← TanStack Table (nome, email, qualifica, contratto, stato)
  StaffModal.tsx                            ← shadcn Dialog — crea/modifica dipendente
  StaffStatusBadge.tsx                      ← badge attivo/inattivo
  StaffSearchFilters.tsx                    ← ricerca testo + filtro qualifica + stato
```

### StaffModal — campi

**Sezione identità**
| Campo | Tipo | Validazione |
|---|---|---|
| nome | text | obbligatorio |
| cognome | text | obbligatorio |
| email | email | obbligatorio, univoca (RF-B CA1) |
| telefono | text | opzionale |

**Sezione contrattuale (admin only)**
| Campo | Tipo | Validazione |
|---|---|---|
| qualificationId | select | obbligatorio |
| contractType | enum: full-time/part-time/contractor | obbligatorio |
| contractHoursWeekly | number | 1..60 |
| attivo | boolean | default true |

### Vincoli di business

- **RF-B CA1:** email duplicata → errore 422 da API → `form.setError('email', ...)`
- **RF-B CA2:** disattivazione → PATCH `{ active: false }`, i turni storici restano; 
  conferma: "Il dipendente non potrà ricevere nuovi turni futuri"
- **RF-B CA3:** qualifica con dipendenti/coperture → non eliminabile (gestito lato API);
  UI mostra link "Riassegna prima" se il DELETE restituisce 409

### Integrazione Zod

Usa `UserForm` da TSK-007 come base; `StaffModal` avvolge il form in un Dialog.

## Criteri di accettazione

- [ ] Crea dipendente con email duplicata → errore inline sul campo email (RF-B CA1)
- [ ] Disattiva dipendente → badge "Inattivo", assenza dalla selezione in MatrixGrid (RF-B CA2)
- [ ] Tabella: filtro per qualifica e testo funzionanti, empty state se risultato vuoto
- [ ] Ordine colonne: cognome, nome, email, qualifica, ore/settimana, contratto, stato
- [ ] Paginazione o scroll virtuale per liste con 50+ dipendenti
- [ ] Modifica: precompila tutti i campi; submit aggiorna la riga senza reload pagina
