## Pattern standard per ogni Route Handler

```typescript
// app/api/shifts/route.ts
export async function POST(req: Request) {
  const session = await auth();
  if (!session) return Response.json({ error: 'unauthorized' }, { status: 401 });
  if (session.user.role !== 'admin')
    return Response.json({ error: 'forbidden' }, { status: 403 });

  const body = await req.json();
  const parsed = shiftCreateSchema.safeParse(body); // Zod вЂ” stesso schema FE
  if (!parsed.success)
    return Response.json({ error: 'validation', issues: parsed.error.issues }, { status: 400 });

  // RB-01..RB-17 check (TSK-006 вЂ” stub per ora)
  // const violations = await validateShift(parsed.data);
  // if (violations.blocking.length > 0) return 400 ...

  const shift = await db.insert(shifts).values(parsed.data).returning();
  await insertAuditLog({ actor: session.user.id, action: 'shift.create', ... });
  return Response.json(shift[0], { status: 201 });
}
```

## Zod schemas condivisi (lib/zod/)
- `shiftCreateSchema` вЂ” validazione input POST /api/shifts (usato anche da FE)
- `requestCreateSchema` вЂ” input POST /api/requests
- `userPatchSchema` вЂ” PATCH /api/users/me (solo campi consentiti, T-SEC-04)

## Criteri di accettazione
- [ ] GET /api/shifts?userId=X con session dipendente в†’ solo turni propri (T-SEC-01)
- [ ] POST /api/shifts con session dipendente в†’ 403 (T-SEC-02)
- [ ] POST /api/requests con payload invalido в†’ 400 con Zod issues
- [ ] PATCH /api/users/me con `qualificationId` в†’ 403 (RB-13, T-SEC-04)
- [ ] PATCH /api/requests/{id}/accept-swap con session non-destinatario в†’ 403 (T-SEC-05)
- [ ] Ogni scrittura genera entry in audit_log
- [ ] Tutti gli endpoint restituiscono 401 con token scaduto (T-SEC-03)
