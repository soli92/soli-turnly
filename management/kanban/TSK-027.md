# TSK-027 — Report straordinari / ore

## Obiettivo

Implementare la schermata "Report straordinari/ore" (RF-I, screen 12 inventario) e il
relativo endpoint API. Il modulo `calculateOvertime` è già presente in
`lib/rules/calculateOvertime.ts` e implementa RB-06.

L'admin seleziona un periodo (from/to), filtra opzionalmente per dipendente, e vede la
tabella ore ordinarie / straordinarie / totale per ogni dipendente.

## Componenti / Endpoint da creare

### Pagina (fe)

```
app/(admin)/reports/
  page.tsx                          ← RSC: verifica ruolo admin; redirect → /reports/overtime
  overtime/
    page.tsx                        ← RSC: fetch overtime data via /api/admin/reports/overtime
    _components/
      OvertimeReportClient.tsx      ← Client: filtri + tabella TanStack Table v8
      OvertimeFilters.tsx           ← DateRangePicker (from/to), select dipendente, pulsante Export CSV
      OvertimeTable.tsx             ← TanStack Table: colonne dipendente, ore contratto, ore ordinarie, ore straordinario, totale, badge soglia
      OvertimeRowBadge.tsx          ← badge rosso "Sopra soglia" se straordinario > maxStraordinarioMensileOre (RF-I CA2)
```

### API (be)

```
GET /api/admin/reports/overtime?from=YYYY-MM-DD&to=YYYY-MM-DD&userId=<uuid|undefined>
```

- Admin only (403 per dipendenti)
- Query: per ogni dipendente nel range (o solo il `userId` specificato):
  - Seleziona tutti i turni `WHERE date BETWEEN from AND to`
  - Calcola ore totali con `calculateOvertime` (RB-06, DST-safe con `lib/date`)
  - Restituisce array ordinato per straordinario desc
- Response:

```typescript
{
  data: Array<{
    userId: string;
    firstName: string;
    lastName: string;
    qualificationName: string | null;
    contractHours: number;           // ore/settimana contratto
    ordinaryHours: number;           // ore nel range (fino a contratto)
    overtimeHours: number;           // ore oltre il contratto
    totalHours: number;
    overtimeExceedsThreshold: boolean; // > maxStraordinarioMensileOre (RB-06)
  }>;
  period: { from: string; to: string };
  generatedAt: string; // ISO timestamp
}
```

- Paginazione: `page` + `limit` (default 50)
- Export CSV: il client genera CSV lato browser da `data[]` (nessun endpoint separato)

## Acceptance Criteria

- [ ] RF-I CA1: dipendente con `contractHours=40` a cui sono assegnate 46h in una settimana → report mostra `overtimeHours=6.00` (T-DOM-06 equivalente report)
- [ ] RF-I CA2: se `overtimeHours > maxStraordinarioMensileOre (40)`, la riga mostra `OvertimeRowBadge` rosso "Sopra soglia"
- [ ] `GET /api/admin/reports/overtime` da dipendente non-admin → 403
- [ ] Filtro `userId` su dipendente specifico → ritorna solo quella riga
- [ ] Filtro date: `from > to` → 400 Bad Request "from deve precedere to"
- [ ] Nessun turno nel periodo → `data: []`, nessun errore
- [ ] Export CSV: click "Esporta CSV" genera file `report_straordinari_<from>_<to>.csv` con header e una riga per dipendente
- [ ] OvertimeTable ordinata per `overtimeHours` decrescente (dipendente con più straordinario in cima)
- [ ] Skeleton di caricamento durante fetch (`isLoading`) — nessun layout shift
- [ ] Layout responsive: tabella con scroll orizzontale su mobile (375px)
