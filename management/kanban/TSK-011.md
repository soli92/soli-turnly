# TSK-011 — Accessibilità WCAG 2.2 AA

## Obiettivo
Eseguire uno scan axe-core su tutte le route dell'applicazione, correggere le
violazioni e mantenere zero-violation come gate di CI per EP-007.

## Setup axe-playwright

```typescript
// tests/a11y/setup.ts
import { checkA11y, injectAxe } from 'axe-playwright';

export async function scanRoute(page: Page, route: string) {
  await page.goto(route);
  await injectAxe(page);
  await checkA11y(page, undefined, {
    axeOptions: {
      runOnly: {
        type: 'tag',
        values: ['wcag2a', 'wcag2aa', 'wcag21aa', 'wcag22aa'],
      },
    },
    detailedReport: true,
    detailedReportOptions: { html: true },
  });
}
```

## Criteri WCAG da verificare (priority issues tipici per griglia)

### Matrice turni (ShiftGrid)
- Table semantica: `<table>` con `<th scope="col">` per giorni, `<th scope="row">` per dipendenti
- O in alternativa `role="grid"` con `role="gridcell"` e `aria-label` su ogni cella
- Focus management: Enter apre editor, Escape chiude, frecce navigano celle
- ViolationBadge: testo alternativo su badge icon (non solo colore)
- Celle vuote: `aria-label="Nessun turno assegnato"` (non cella vuota)

### Form (ShiftEditor, RequestForm)
- Ogni `<input>` ha `<label>` associata o `aria-label`
- Errori di validazione: `aria-describedby` collega input al messaggio errore
- Dialog: `aria-modal="true"`, focus intrappolato dentro (shadcn Dialog già lo fa)
- Date range picker: accessibile da tastiera

### Contrasto colori
- Badge stati: tutti ≥ 4.5:1 su sfondo bianco (già corretti nel prototipo)
- Testo su btn-primary (#2563eb): bianco su blu → 8.59:1 ✓
- Stato "Riposo": testo grigio su bianco ≥ 4.5:1

### Navigazione
- Skip link: `<a href="#main-content">Salta al contenuto</a>` visibile on focus
- Sidebar: `<nav aria-label="Menu principale">`, item attivo con `aria-current="page"`
- Tutte le azioni da mouse raggiungibili da tastiera
