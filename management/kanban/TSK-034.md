# TSK-034 — Inngest Cloud setup produzione

## Obiettivo

Creare l'account Inngest Cloud e ottenere le chiavi di produzione
(`INNGEST_EVENT_KEY` e `INNGEST_SIGNING_KEY`) necessarie per i background
job (audit log, notifiche email, ricorrenze).

## Step

### 1. Creare account Inngest Cloud

- Accedere a [app.inngest.com](https://app.inngest.com) → Sign up (gratis)
- Creare un'organizzazione: `soli-turnly`

### 2. Creare l'app

- Dashboard → Apps → New App
- Nome: `turnly-production`
- Environment: `Production`

### 3. Recuperare le chiavi

Dalla dashboard dell'app:
- **Event Key** → `INNGEST_EVENT_KEY`
- **Signing Key** → `INNGEST_SIGNING_KEY`

Inserire entrambe in Vercel (TSK-033) prima di avviare il primo deploy.

### 4. Registrare le funzioni (post-deploy)

Dopo il primo deploy Vercel (TSK-033), sincronizzare le funzioni Inngest:

```
Inngest Dashboard → Apps → turnly-production → Sync
URL: https://<nome-progetto>.vercel.app/api/inngest
```

Verificare che le funzioni definite in `lib/inngest/` compaiano nella dashboard.

## Acceptance Criteria

- [ ] Account Inngest Cloud creato
- [ ] `INNGEST_EVENT_KEY` e `INNGEST_SIGNING_KEY` disponibili
- [ ] Chiavi inserite nelle env vars Vercel (TSK-033)
- [ ] Post-deploy: funzioni Inngest sincronizzate e visibili in dashboard
