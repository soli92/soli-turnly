# TSK-003 — Autenticazione e autorizzazione

## Obiettivo
Implementare autenticazione email/password con Auth.js v5, gestione sessione JWT
con scadenza configurabile, e middleware RBAC per le route protette.

## Scope

### 1. Auth.js v5 config (auth.ts)
```typescript
import NextAuth from 'next-auth';
import Credentials from 'next-auth/providers/credentials';
import { DrizzleAdapter } from '@auth/drizzle-adapter';
import bcrypt from 'bcryptjs';

export const { auth, handlers, signIn, signOut } = NextAuth({
  adapter: DrizzleAdapter(db),
  session: { strategy: 'jwt', maxAge: parseInt(process.env.SESSION_MAX_AGE ?? '28800') },
  providers: [
    Credentials({
      credentials: { email: {}, password: {} },
      authorize: async ({ email, password }) => {
        const user = await db.query.users.findFirst({
          where: eq(users.email, email as string)
        });
        if (!user) return null; // RF-A CA1: messaggio generico
        const valid = await bcrypt.compare(password as string, user.passwordHash);
        if (!valid) return null;
        return { id: user.id, role: user.role, email: user.email };
      }
    })
  ],
  callbacks: {
    jwt: ({ token, user }) => { if (user) token.role = user.role; return token; },
    session: ({ session, token }) => { session.user.role = token.role; return session; }
  }
});
```

### 2. Middleware RBAC (middleware.ts)
```typescript
export default auth((req) => {
  const { pathname } = req.nextUrl;
  const role = req.auth?.user?.role;

  if (pathname.startsWith('/admin') && role !== 'admin')
    return Response.json({ error: 'forbidden' }, { status: 403 });

  if (pathname.startsWith('/api/admin') && role !== 'admin')
    return Response.json({ error: 'forbidden' }, { status: 403 });

  if (!req.auth && !pathname.startsWith('/login'))
    return Response.redirect(new URL('/login', req.url));
});
```

### 3. Pagine
- `app/(auth)/login/page.tsx` — form email/password (shadcn/ui)
- Errore login: messaggio generico "Credenziali non valide" (RF-A CA1)

### 4. Cambio password (dipendente)
- Route `PATCH /api/users/me/password` — verifica old password, bcrypt nuovo hash

## Criteri di accettazione
- [ ] Login con credenziali valide → redirect a /dashboard (admin) o /calendar (employee)
- [ ] Login fallito → messaggio generico (non rivela quale campo errato — RF-A CA1)
- [ ] Dipendente accede a /admin/* → 403 (RF-A CA2) — T-SEC-02
- [ ] Token scaduto → API risponde 401, client redirect /login (RF-A CA3) — T-SEC-03
- [ ] `SESSION_MAX_AGE` configurabile via env (default 28800 = 8h)
- [ ] Cambio password: vecchia errata → 422, nuova salvata correttamente → 200
