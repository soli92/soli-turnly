# TSK-020 — Coda richieste/approvazioni page

## Obiettivo

Implementare la pagina di gestione delle richieste lato admin (RF-M admin side, F6/F7/F8):
inbox filtrabile, dettaglio richiesta con anteprima impatto, azioni approva/rifiuta con
rivalidazione RB-14, e gestione stato `in_attesa_collega` per gli scambi.

## Componenti da creare

```
app/(admin)/requests/page.tsx               ← RSC: lista richieste pending
app/(admin)/requests/[id]/page.tsx          ← CSR: dettaglio + approval panel
components/requests/
  RequestQueue.tsx                          ← TanStack Table: tipo, dipendente, data, stato
  RequestQueueFilters.tsx                   ← filtri: tipo (assenza/scambio/turno/modifica), stato
  RequestStatusBadge.tsx                    ← pill stato con colore
  RequestDetail.tsx                         ← scheda dettaglio richiesta
  ApprovalImpactPanel.tsx                   ← pannello anteprima violazioni + warnings
  ApprovalActions.tsx                       ← pulsanti Approva / Rifiuta + motivo rifiuto
  SwapColleagueStatus.tsx                   ← stato accettazione collega (RF-M, RF-F CA3)
```

### Approval flow (RB-14)

```typescript
// Approva → POST /api/requests/:id/approve
// BE esegue rivalidazione RB-14 (riapplica RB-01..08 allo stato corrente)
// Se violazione bloccante:
//   → 409 { blocking: [{ ruleId, message }] }
//   → UI: "Approvazione bloccata — motivo: RB-01 sovrapposizione"
//   → pulsante Approva rimane disabilitato
// Se solo warnings o nessuno:
//   → 200, toast "Richiesta approvata"
//   → invalida query ['requests']
```

### ApprovalImpactPanel

```typescript
// GET /api/requests/:id/impact
// Calcola impatto sul planning prima dell'approvazione:
// - assenza: giorni bloccati + turni in conflitto
// - scambio: RB-10 per entrambe le parti
// - nuovo turno: RB-01..08 per il dipendente
// Mostra: { blocking: [], warnings: [], summary: string }
```

### Stato `in_attesa_collega` (RF-M, RF-F CA3)

- Richieste scambio con stato `in_attesa_collega`: row con badge + tooltip "In attesa dell'accettazione di [Collega]"
- `SwapColleagueStatus` mostra: collega, data proposta, azioni non disponibili all'admin
- Quando collega accetta → SSE `swap.accepted` → query invalidata → stato aggiornato

## Criteri di accettazione

- [ ] Inbox: richieste con stato `sent` e `in_attesa_collega` visibili, altre filtrabili
- [ ] Approvazione con violazione bloccante RB-14 → 409 + messaggio esplicativo (T-REQ-02)
- [ ] Approvazione richiesta assenza → assenza attiva creata (T-REQ-01)
- [ ] Approvazione scambio che supera RB-10 → swap eseguito, notifiche a entrambe le parti
- [ ] Rifiuto: motivo obbligatorio (textarea, max 500 chars); 200 → toast + riga aggiornata
- [ ] Richiesta applicata: readonly, nessuna azione disponibile (RB-16, T-REQ-04)
- [ ] InboxBadge su dashboard aggiornato dopo ogni approvazione/rifiuto
