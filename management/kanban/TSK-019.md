# TSK-019 — Ricorrenze/cicli wizard page

## Obiettivo

Implementare la pagina di definizione e generazione delle ricorrenze e dei cicli rotativi
(RF-E, F2): wizard a 3 step con anteprima dei turni generati, report conflitti,
e triggering del job Inngest `generateRecurringShifts` (TSK-009).

## Componenti da creare

```
app/(admin)/recurrence/page.tsx             ← RSC: lista ricorrenze esistenti
app/(admin)/recurrence/new/page.tsx         ← CSR: wizard
components/recurrence/
  RecurrenceList.tsx                        ← lista ricorrenze attive con azioni
  RecurrenceWizard.tsx                      ← orchestratore 3 step con stepper UI
  step1-RecurrenceTypeStep.tsx              ← tipo: settimanale | ciclo rotativo
  step2-RecurrenceSequenceStep.tsx          ← configurazione sequenza/target/date
  step3-RecurrencePreviewStep.tsx           ← anteprima turni generati + report conflitti
  RecurrenceConflictReport.tsx              ← lista conflitti con conteggio salti (RB-11)
```

### Step 1 — Tipo ricorrenza

- **Settimanale:** selezione giorni della settimana + tipologia turno per giorno
- **Ciclo rotativo:** sequenza di tipologie (es: Mattina, Notte, Riposo, Mattina...) con lunghezza ciclo e offset per dipendente

### Step 2 — Configurazione

- Target dipendenti: multi-select con filtro qualifica
- Intervallo date: `startDate` + `endDate` (data picker)
- Festivi: toggle "Salta festivi" (con lista festivi configurabili in Settings)
- Validazione: `endDate > startDate`, almeno 1 dipendente selezionato

### Step 3 — Anteprima + report

```typescript
// POST /api/admin/recurrence/preview (dry-run)

```

### Generazione

```typescript
// "Genera" → POST /api/admin/recurrence/generate
// → emette evento Inngest 'shift.recurrence.trigger' (TSK-009)
```

### Modifica occorrenza singola (RF-E CA2)

- Dalla lista, click su ricorrenza → elenco occorrenze con toggle "Modifica solo questa" / "Modifica serie"
- Modifica serie: propaga solo alle future non personalizzate

## Criteri di accettazione

- [ ] Wizard step 1→2→3: navigazione forward/backward mantiene i dati (no reset)
- [ ] Step 3 preview: ricorrenza con assenza approvata → occorrenza mostrata come "Saltata (assenza)" (RF-E CA1, RB-11, T-REC-01)
- [ ] Generazione: N turni creati correttamente; job Inngest completato (T-REC-01)
- [ ] Modifica singola occorrenza → le altre occorrenze non cambiano (RF-E CA2, T-REC-02)
- [ ] Preview mostra report conflitti collassabile; "Genera" disabilitato finché preview non caricata
- [ ] Lista ricorrenze: azioni Modifica, Disattiva, Visualizza occorrenze
