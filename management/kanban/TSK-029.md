# TSK-029 — Email notification templates + Inngest dispatch

## Obiettivo

Sostituire i template HTML inline string di `lib/jobs/sendNotificationEmail.ts` con
componenti React Email strutturati. Collegare il dispatch dell'evento Inngest
`notification/email.send` agli handler API che creano notifiche in-app, in modo che
ogni notifica in-app opzionalmente attivi anche l'email.

Il job Inngest `sendNotificationEmail` è già registrato in `app/api/inngest/route.ts`.
Il provider di invio (Resend) è già configurato via `RESEND_API_KEY`.

## Componenti / File da creare o modificare

### Template React Email (be)

```
lib/email-templates/
  base-layout.tsx          ← layout comune: logo Turnly, footer, link unsubscribe
  ShiftAssignedEmail.tsx   ← template "turno assegnato" (sostituisce case 'shift-assigned')
  RequestApprovedEmail.tsx ← template "richiesta approvata"
  RequestRejectedEmail.tsx ← template "richiesta rifiutata"
  SwapRequestEmail.tsx     ← template "proposta scambio ricevuta"
  index.ts                 ← barrel export
```

Ogni template è un React component che riceve props tipizzate:

```typescript
// Esempio
interface ShiftAssignedEmailProps {
  recipientName: string;
  date: string;           // es. "Lunedì 15 luglio 2026"
  startTime: string;      // es. "08:00"
  endTime: string;        // es. "16:00"
  shiftTypeName: string;
  appUrl: string;
}
```

I componenti usano `@react-email/components` (`Html`, `Body`, `Container`, `Text`, `Button`,
`Hr`, `Head`, `Preview`). Il `buildEmailHtml` in `sendNotificationEmail.ts` viene sostituito
da `render(component, { pretty: false })` di `@react-email/render`.

### Aggiornamento job (be)

```
lib/jobs/sendNotificationEmail.ts  ← sostituire buildEmailHtml con render(template)
```

```typescript
import { render } from '@react-email/render';
import { ShiftAssignedEmail } from '@/lib/email-templates';

// Dentro step.run:
const html = render(<ShiftAssignedEmail {...typedData} />);
```

### Dispatch Inngest da API handlers (be)

Aggiungere dispatch `inngest.send('notification/email.send', {...})` agli handler che
creano notifiche in-app quando `canaleNotifiche` include email:

```
app/api/requests/[id]/approve/route.ts   ← dopo salvataggio: dispatch 'request-approved' al richiedente
app/api/requests/[id]/reject/route.ts    ← dispatch 'request-rejected' al richiedente
app/api/requests/[id]/accept-swap/route.ts ← dispatch 'swap-request' al collega bersaglio
app/api/shifts/route.ts                  ← dopo POST (assegnazione): dispatch 'shift-assigned' al dipendente
```

Condizione: il dispatch avviene solo se l'utente destinatario ha un'email valida nel DB.
Non bloccare la risposta API se il dispatch fallisce (fire-and-forget).

### Variabili d'ambiente necessarie

```
RESEND_API_KEY        — già configurato
RESEND_FROM_EMAIL     — default noreply@turnly.app
NEXT_PUBLIC_APP_URL   — usato per link nel template
```

## Acceptance Criteria

- [ ] `npm run build` non produce errori con le nuove dipendenze `@react-email/components` e `@react-email/render`
- [ ] `render(<ShiftAssignedEmail {...} />)` restituisce HTML valido (non vuoto, contiene il nome destinatario)
- [ ] Con `RESEND_API_KEY` non impostata: il job logga su console e ritorna `{ provider: 'stub' }` senza lanciare eccezioni
- [ ] Con `RESEND_API_KEY` impostata: il job chiama `resend.emails.send()` con `html` generato da React Email
- [ ] Dopo approvazione di una richiesta (`POST /api/requests/[id]/approve`): viene emesso evento `notification/email.send` con `template='request-approved'` e `to = email del richiedente` (verificabile in Inngest dev server o mock)
- [ ] Dopo rifiuto di una richiesta: evento `request-rejected` emesso
- [ ] Dopo assegnazione turno (`POST /api/shifts`): evento `shift-assigned` emesso al dipendente assegnato
- [ ] Dispatch non bloccante: se Inngest `send()` lancia eccezione, la risposta API non è compromessa (try/catch + log)
- [ ] Template `SwapRequestEmail`: contiene il nome del richiedente, il turno offerto, il turno del destinatario, un link all'app per accettare/rifiutare
- [ ] Tutti i template passano `render()` senza prop mancanti quando si usano le interfacce definite
