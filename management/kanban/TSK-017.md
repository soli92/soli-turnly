# TSK-017 — Gestione assenze page

## Obiettivo

Implementare la pagina di gestione assenze (RF-G, F4): registrazione di ferie/malattia/
permesso per un dipendente con rilevamento automatico dei turni in conflitto e workflow
di risoluzione prima del salvataggio definitivo.

## Componenti da creare

```
app/(admin)/absences/page.tsx               ← RSC: lista assenze attive + form
components/absences/
  AbsenceTable.tsx                           ← lista: dipendente, tipo, periodo, stato
  AbsenceForm.tsx                            ← form registrazione (integra da TSK-007)
  ConflictShiftList.tsx                      ← lista turni in conflitto con azioni
  AbsenceConflictModal.tsx                   ← Dialog: "Trovati N turni — scegli azione"
  AbsenceTypeBadge.tsx                       ← pill ferie/malattia/permesso
```

### Flusso di registrazione (F4)

```
Admin compila AbsenceForm (dipendente + tipo + date range)
  → onSubmit: POST /api/admin/absences/check-conflicts  (dry-run)
    → se turni in conflitto:  apre AbsenceConflictModal
       → per ogni turno: radio "Annulla" | "Mantieni" | "Riassegna"
       → "Riassegna" apre ShiftEditor inline (TSK-005)
    → se nessun conflitto: POST /api/admin/absences direttamente
  → toast "Assenza registrata"
  → invalidate query ['absences', 'shifts']
```

### API assenze

- `GET /api/admin/absences?userId=X&from=Y&to=Z` — lista con filtri
- `POST /api/admin/absences/check-conflicts` — body: `{ userId, startDate, endDate }` → `{ shifts: ShiftConflict[] }`
- `POST /api/admin/absences` — body include `conflictResolutions: { shiftId, action }[]`
- `DELETE /api/admin/absences/:id` — rimuove assenza (se non ancora applicata)

### Vincoli (RF-G)

- **RF-G CA1:** dopo registrazione, RB-08 impedisce nuovi turni nelle date (verificato in TSK-006/TSK-004)
- **RF-G CA2:** i turni in conflitto non vengono eliminati silenziosamente — sempre mostrati con azioni

## Criteri di accettazione

- [ ] Registrazione assenza senza conflitti → salvataggio diretto, toast, riga in tabella (RF-G)
- [ ] Registrazione assenza con 3 turni in conflitto → AbsenceConflictModal mostra i 3 turni
- [ ] Turno con azione "Annulla" → turno rimosso dopo conferma (RF-G CA2)
- [ ] Assenza registrata → assegnazione turno nelle stesse date bloccata in MatrixGrid (RF-G CA1, RB-08)
- [ ] AbsenceForm: date range end ≥ start, dipendente obbligatorio, tipo obbligatorio
- [ ] Lista assenze: filtro per dipendente + periodo; empty state se nessuna assenza attiva
