# TSK-022 — Le mie richieste page

## Obiettivo

Implementare la pagina "Le mie richieste" del dipendente (RF-M employee side): lista
delle proprie richieste con stato, cronologia decisioni, azioni di annullamento, e
gestione speciale per le proposte di scambio ricevute da un collega.

## Componenti da creare

```
app/(employee)/requests/page.tsx            ← RSC: fetch richieste del dipendente autenticato
app/(employee)/requests/[id]/page.tsx       ← CSR: dettaglio richiesta
components/employee/requests/
  MyRequestList.tsx                         ← lista con filtri tipo/stato
  MyRequestCard.tsx                         ← card richiesta (tipo, data, stato, azioni)
  RequestTimeline.tsx                       ← cronologia: inviata → in attesa → decisione
  RequestCancelButton.tsx                   ← annulla (solo se stato consentito, RB-16)
  SwapReceivedCard.tsx                      ← card speciale per scambio ricevuto da collega
  SwapAcceptRejectPanel.tsx                 ← accetta/rifiuta scambio con dettaglio turni
```

### Lista richieste

- Fonte: `GET /api/requests?mine=true` (filtra per `userId = session.user.id`)
- Colonne/card: tipo (badge), data invio, periodo richiesto, stato (badge colorato), azioni
- Filtri: per tipo (assenza / scambio / nuovo turno / modifica), per stato
- Ordinamento default: più recenti prima

## Criteri di accettazione

- [ ] Lista mostra solo le richieste del dipendente autenticato (RF-M CA3, T-SEC-07)
- [ ] Annulla richiesta `sent` → stato cambia a `cancelled`, pulsante scompare (RF-M CA5)
- [ ] Tentativo annullamento richiesta `applied` → errore visibile "Non più annullabile" (RB-16, T-REQ-04)
- [ ] SwapReceivedCard: visibile solo se session.user.id === request.colleagueId (T-SEC-08)
- [ ] Accetta scambio → notifica all'admin; stato aggiornato live via SSE
- [ ] Notifica approvazione/rifiuto → badge stato aggiornato entro 30s senza refresh
- [ ] Empty state se nessuna richiesta; skeleton durante fetch
