# TSK-033 — Setup progetto Vercel

## Obiettivo

Configurare il progetto su Vercel collegato al repository GitHub, impostare
la directory radice corretta e tutte le variabili d'ambiente di produzione.
Il deploy automatico su push a `main` è incluso nativamente nell'integrazione
GitHub di Vercel — non serve un workflow CD separato.

## Step

### 1. Importare il repository su Vercel

- Accedere a [vercel.com](https://vercel.com) → Add New Project → Import Git Repository
- Selezionare `soli-turnly`
- **Root Directory**: impostare `code/app` ← critico, senza questo il build fallisce
- Framework: Next.js (rilevato automaticamente)
- Non avviare il deploy ancora (mancano le env vars)

### 2. Configurare le variabili d'ambiente (Production)

| Variabile | Valore | Fonte |
|---|---|---|
| `DATABASE_URL` | URL pooled Neon | TSK-032 |
| `NEXTAUTH_SECRET` | stringa random 32 byte | `openssl rand -base64 32` |
| `NEXTAUTH_URL` | `https://<nome-progetto>.vercel.app` | Vercel dashboard |
| `INNGEST_EVENT_KEY` | event key Inngest cloud | TSK-034 |
| `INNGEST_SIGNING_KEY` | signing key Inngest cloud | TSK-034 |
| `NODE_ENV` | `production` | — |

> `NEXTAUTH_URL` deve essere aggiornato se si configura un dominio custom.

### 3. Verificare l'integrazione GitHub

- Settings → Git → verificare che `main` sia il branch di produzione
- Preview deployments attivi per le PR (default)
- Nessun workflow GitHub Actions aggiuntivo necessario per il CD

### 4. Avviare il primo deploy

- Vercel dashboard → Deployments → Redeploy (oppure push vuoto su main)
- Attendere il build — verificare che non ci siano errori nel log

## Acceptance Criteria

- [ ] Progetto Vercel importato con Root Directory = `code/app`
- [ ] Tutte le 6 variabili d'ambiente configurate in Production
- [ ] Primo build completato senza errori
- [ ] URL di produzione raggiungibile (risponde con la login page)
- [ ] Push su `main` trigghera automaticamente un nuovo deploy
