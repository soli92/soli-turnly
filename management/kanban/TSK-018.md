# TSK-018 — Fabbisogni/copertura page

## Obiettivo

Implementare la pagina di gestione dei fabbisogni di copertura (RF-H, RB-07): tab setup
per definire i minimi per qualifica/fascia e tab monitor per visualizzare le sotto-coperture
per giorno e fascia con scarti.

## Componenti da creare

```
app/(admin)/coverage/page.tsx              ← Tabs: "Setup fabbisogni" | "Monitor copertura"
components/coverage/
  CoverageRuleTable.tsx                    ← lista regole: qualifica, tipologia, giorno, minimo
  CoverageRuleModal.tsx                    ← Dialog CRUD regola (integra CoverageForm TSK-007)
  CoverageMonitorGrid.tsx                  ← griglia giorni × fasce con indicatori copertura
  CoverageCell.tsx                         ← cella: pianificati/minimi, colore verde/rosso/giallo
  CoverageLegend.tsx                       ← legenda colori
```

### Tab Setup fabbisogni

- Lista `coverage_requirements` con TanStack Table
- Azioni: crea, modifica, elimina (solo se non ci sono turni attivi nella fascia)
- Colonne: Qualifica, Tipologia turno (o "Tutte"), Giorno settimana (o "Tutti"), Minimo

### Tab Monitor copertura

```typescript
// Dati: joins shifts + coverage_requirements per il periodo selezionato
// GET /api/admin/coverage/monitor?from=ISO&to=ISO
// Response: { date: ISO, shiftTypeId: string, qualificationId: string,
//             required: number, actual: number, deficit: number }[]

// Visualizzazione: griglia data × fascia
// Verde: actual >= required
// Giallo: actual >= required - 1 (quasi copertura)
// Rosso: actual < required (sotto-coperta) → mostra deficit (-N)
```

### Aggiornamento live (RF-H CA2)

```typescript
// Invalida query 'coverage-monitor' quando:
// 1. Si aggiunge/rimuove un turno (evento SSE shift.assigned / shift.modified)
// 2. Si cambia il selettore periodo
useNotifications(); // TSK-008 SSE
```

## Criteri di accettazione

- [ ] Creazione regola: fascia notte 3 Infermieri → appare nel monitor come "3 richiesti"
- [ ] Monitor: fascia con 2 Infermieri pianificati su minimo 3 → cella rossa con "-1" (RF-H CA1, T-DOM-07)
- [ ] Monitor: dopo aggiunta turno mancante → cella diventa verde senza refresh (RF-H CA2)
- [ ] Setup: elimina regola → dialog conferma, se fascia ha turni pianificati oggi mostra avviso
- [ ] Navigazione periodo: selezione settimana/mese aggiorna il monitor
- [ ] Empty state: nessuna regola configurata → messaggio "Nessun fabbisogno definito"
