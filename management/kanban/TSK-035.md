# TSK-035 — Smoke test primo rilascio produzione

## Obiettivo

Verificare manualmente i flussi core dell'applicazione sull'URL di produzione
Vercel dopo il primo deploy. Nessun script automatico — test esplorativo sul
golden path admin + dipendente.

## Flussi da verificare

### Admin

- [ ] Login con le credenziali admin del seed (TSK-032)
- [ ] Dashboard Admin: KPI visibili, nessun errore console
- [ ] Anagrafica dipendenti: lista caricata, creazione nuovo dipendente OK
- [ ] Tipologie di turno: lista caricata, creazione nuova tipologia OK
- [ ] Matrice turni: griglia caricata, assegnazione turno a dipendente OK
- [ ] Gestione assenze: registrazione assenza → nessun errore
- [ ] Fabbisogni copertura: pagina caricata correttamente
- [ ] Report straordinari: pagina caricata con dati

### Dipendente

- [ ] Login con un utente dipendente (creato dal seed o dall'admin)
- [ ] Calendario: turni visibili per il mese corrente
- [ ] Le mie richieste: lista caricata
- [ ] Nuova richiesta: apertura form multi-step OK

### Infra

- [ ] Nessun errore 500 nei log Vercel durante i test
- [ ] Inngest dashboard: nessun job in errore
- [ ] `NEXTAUTH_URL` corrisponde all'URL visitato (no redirect loop)

## Definition of Done

- Tutti i check sopra spuntati
- Screenshot o note sugli eventuali comportamenti anomali da aprire come issue
- Release 1.0 dichiarata **live** in `management/roadmap.md` con data e URL prod
