# TSK-008 — Layer notifiche real-time (SSE)

## Obiettivo
Implementare la distribuzione di notifiche server→client via Server-Sent Events,
senza WebSocket. Gestire connessioni persistenti, fan-out per userId e recovery
su disconnessione.

## Architettura SSE

```
Route Handler (GET /api/notifications/sse)
    ↓ ReadableStream → Response con content-type text/event-stream
    
Server writes:
    lib/sse/broker.ts → Map<userId, Set<WritableStreamDefaultWriter>>
    
Trigger events:
    lib/sse/emit.ts → emitToUser(userId, event)
    
Client:
    hooks/useNotifications.ts → new EventSource('/api/notifications/sse')
```

## Implementazione (app/api/notifications/sse/route.ts)

```typescript
import { auth } from '@/auth';
import { broker } from '@/lib/sse/broker';

export const dynamic = 'force-dynamic';

export async function GET() {
  const session = await auth();
  if (!session) return new Response('Unauthorized', { status: 401 });

  const userId = session.user.id;
  let controller: ReadableStreamDefaultController;

  const stream = new ReadableStream({
    start(c) {
      controller = c;
      broker.addClient(userId, controller);
      // heartbeat ogni 30s per tenere la connessione viva
      const hb = setInterval(() => {
        try { controller.enqueue(': heartbeat\n\n'); } catch { clearInterval(hb); }
      }, 30_000);
    },
    cancel() {
      broker.removeClient(userId, controller);
    },
  });

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
      'Connection': 'keep-alive',
    },
  });
}
```

## Broker (lib/sse/broker.ts)

```typescript
const clients = new Map<string, Set<ReadableStreamDefaultController>>();

export const broker = {
  addClient(userId: string, ctrl: ReadableStreamDefaultController) {
    if (!clients.has(userId)) clients.set(userId, new Set());
    clients.get(userId)!.add(ctrl);
  },
  removeClient(userId: string, ctrl: ReadableStreamDefaultController) {
    clients.get(userId)?.delete(ctrl);
  },
  emit(userId: string, event: { type: string; data: unknown }) {
    const payload = `data: ${JSON.stringify(event)}\n\n`;
    clients.get(userId)?.forEach(ctrl => {
      try { ctrl.enqueue(payload); } catch { /* client disconnected */ }
    });
  },
};
```

## Tipi di eventi emessi

| Event type | Trigger | Destinatario |
|---|---|---|
| `shift.assigned` | Admin assegna turno | Dipendente |
| `shift.modified` | Admin modifica turno | Dipendente |
| `request.received` | Dipendente invia richiesta | Admin (tutti) |
| `request.approved` | Admin approva | Dipendente richiedente |
| `request.rejected` | Admin rifiuta | Dipendente richiedente |
| `swap.request` | Dipendente propone scambio | Collega destinatario |
| `swap.accepted` | Collega accetta | Richiedente + Admin |

## Client hook (hooks/useNotifications.ts)

```typescript
export function useNotifications() {
  const { data: session } = useSession();
  const queryClient = useQueryClient();

  useEffect(() => {
    if (!session) return;
    const es = new EventSource('/api/notifications/sse');
    es.onmessage = (e) => {
      const event = JSON.parse(e.data);
      if (event.type === 'shift.assigned') {
        queryClient.invalidateQueries({ queryKey: ['shifts'] });
        toast.info('Nuovo turno assegnato');
      }
      // ... altri tipi
    };
    es.onerror = () => { es.close(); /* reconnect gestito automaticamente dal browser */ };
    return () => es.close();
  }, [session, queryClient]);
}
```

## Criteri di accettazione
- [ ] GET /api/notifications/sse risponde con `content-type: text/event-stream`
- [ ] Connessione emette heartbeat ogni 30s (nessun timeout proxy)
- [ ] Evento `request.received` arriva in < 1s all'admin dopo invio richiesta dipendente
- [ ] Evento `request.approved` arriva in < 1s al dipendente dopo approvazione admin
- [ ] Disconnessione client → broker rimuove il controller (no memory leak)
- [ ] Multi-tab: stessa notifica arriva su tutte le tab aperte dello stesso userId
- [ ] 401 su SSE endpoint senza sessione (T-SEC-03)
