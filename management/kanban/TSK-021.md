# TSK-021 — Calendario dipendente page

## Obiettivo

Implementare la vista turni personale del dipendente (RF-J, F5): calendario con React
Big Calendar nelle viste giorno/settimana/mese, riepilogo ore, export .ics. Il dipendente
vede esclusivamente i propri turni (T-SEC-01/02).

## Componenti da creare

```
app/(employee)/calendar/page.tsx            ← RSC: fetch turni dipendente autenticato
components/employee/calendar/
  EmployeeCalendar.tsx                      ← React Big Calendar con custom toolbar
  ShiftEvent.tsx                            ← evento custom colorato (colore tipologia)
  ShiftDetailDrawer.tsx                     ← drawer lato: dettaglio turno sola lettura
  CalendarToolbar.tsx                       ← navigazione + toggle viste + export .ics
  HoursSummaryBar.tsx                       ← riepilogo ore ordinarie/straordinario del periodo
  ShiftDetailActions.tsx                    ← azioni: "Richiedi modifica" / "Proponi scambio"
```

### Configurazione React Big Calendar

```typescript
import { Calendar, dateFnsLocalizer } from 'react-big-calendar';
import { format, parse, startOfWeek, getDay } from 'date-fns';
import { it } from 'date-fns/locale';

const localizer = dateFnsLocalizer({
  format, parse, startOfWeek: () => startOfWeek(new Date(), { weekStartsOn: 1 }),
  getDay, locales: { it },
});

// Evento custom
const ShiftEvent = ({ event }: { event: ShiftCalendarEvent }) => (
  <div style={{ backgroundColor: event.color }} className="rounded px-1 text-white text-xs">
    {event.typeName} {format(event.start, 'HH:mm')}–{format(event.end, 'HH:mm')}
  </div>
);
```

### Sicurezza (T-SEC-01/02)

```typescript
// page.tsx (RSC)
const session = await auth();
// La query Drizzle filtra SEMPRE per userId = session.user.id
const shifts = await db.query.shifts.findMany({
  where: and(eq(shifts.userId, session.user.id), between(shifts.date, from, to)),
  with: { shiftType: true },
});
// Non espone userId nell'URL — l'ID viene dal token, non dai parametri
```

## Criteri di accettazione

- [ ] Dipendente vede solo i propri turni — GET /api/shifts con userId di altro dipendente → 403 (T-SEC-01)
- [ ] Viste giorno/settimana/mese: navigazione con frecce + salto a "Oggi" funzionanti
- [ ] ShiftEvent: colore coerente con tipologia, orari leggibili anche in vista mese
- [ ] ShiftDetailDrawer: orari, tipologia, note in sola lettura; nessun controllo di scrittura (RF-J CA2)
- [ ] "Richiedi modifica" → naviga a nuova richiesta pre-compilata con shiftId
- [ ] Export .ics: file scaricato con eventi corretti nel periodo selezionato
- [ ] HoursSummaryBar: dipendente 40h/sett con 46h pianificate → mostra 6h straordinario (T-DOM-06)
- [ ] Layout mobile: vista giorno usabile su 375px; scroll verticale per turni sovrapposti
